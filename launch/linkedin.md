# LinkedIn Post

Launching polyresearch -- distributed autoresearch for multiple machines, multiple contributors, verified results.

Karpathy's autoresearch showed that one AI agent can improve a language model while you sleep. Also works for other hard problems. Give it a metric, let it experiment, keep what works. But research has never been a single-player game. More experiments, more people contributing ideas -- that's how you get better results.  

polyresearch fixes this. Point as many machines as you have at the same problem. Each one runs experiments, and the results are independently verified before anything gets merged. Works with any agent, any model, any language.

If you have something measurable you want to improve -- code performance, model accuracy, agent ability -- you can throw a hundred machines at it and wake up to verified, reproducible progress.

We've been using it internally to improve our own products. Today we're releasing it as open source.  
Ships with some examples:  

- ESLint: single-file linting 24% faster over 75 experiments  
- Core War: evolved a Redcode warrior with 218% improvement over 27 experiments  
- PostCSS: ~20% performance improvement
