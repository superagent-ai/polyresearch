# X Launch Thread

## Post 1

Introducing polyresearch -- distributed autoresearch for multiple machines, multiple contributors, verified results.

One agent on one machine is cool. A hundred agents on a hundred machines working together is better.

## Post 2

Research has never been a single-player game. More experiments in less time, more people contributing ideas -- that's how you get better results and more exciting findings. Always has been.

We built polyresearch to bring that to AI agents.

## Post 3

Based on @karpathy's autoresearch -- same core loop, extended with ideas borrowed from how science already works: peer review, reproducibility, shared experiment logs, and independent verification.

We've been using it internally to improve our own products. Today we're open-sourcing it.

## Post 4

What polyresearch adds to the loop:

- Any number of machines, any number of people, same experiment log
- Every attempt stays -- accepted, failed, crashed. No lost results
- Peer reviewers rerun evals. Contributors can't grade their own homework
